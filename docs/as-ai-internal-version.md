# 내부용 문서: AI 기반 충전기 AS 사업 실행안

## 1. 내부 목표
- 90일 내 파일럿 KPI 개선을 증명해 유료 전환
- 6개월 내 레퍼런스 3건 확보

## 2. 운영 가설
- RAG 코파일럿이 오접수 및 재출동을 유의미하게 줄인다.
- 콜센터 1차 기술응대 자동화가 야간 인력 부담을 줄인다.
- 출동 전 부품 추천이 미지참 재방문을 줄인다.

## 3. 실행 범위
### 3.1 코어
- 증상/에러코드 기반 추천
- 원인 Top 1~3 + 조치 + 부품 + 근거
- 콜센터/현장 공통 워크플로우

### 3.2 후속 모듈
- 자재 판매 연계
- 전문가 풀 매칭
- 기술콜센터 운영
- 점검관리 SaaS

## 4. KPI 운영 체계
- 일간: 질의건수, 낮은 신뢰도 응답 비율
- 주간: 오접수율, 재출동률, FTFR, MTTR
- 월간: 절감액, 1차 해결률, 재방문율

## 5. 데이터 정책
- 접수/조치 텍스트 표준화
- 에러코드/부품명 사전 관리
- 피드백(해결/미해결) 필수 수집

## 6. 조직 운영
- PM: 일정/범위/성과 관리
- 데이터: 정제, 지식베이스 품질관리
- 엔지니어링: 검색/응답 품질 개선, 시스템 연동
- 사업개발: 파일럿 계약, ROI 제시, 전환

## 7. 리스크 관리
- 모델 환각: 근거 없는 답변 차단 규칙
- 책임 이슈: 최종 승인 주체 명시
- 보안 이슈: 최소권한/접근로그/분리 저장
- 확장 과속: 파일럿 KPI 달성 전 모듈 확장 금지

## 8. 내부 의사결정 게이트
- Gate 1: 파일럿 30일, 데이터 품질/사용률 확인
- Gate 2: 파일럿 60일, KPI 개선 추세 확인
- Gate 3: 파일럿 90일, 유료 전환 여부 결정

## 9. 상용 전환 기준 (예시)
- 오접수율 15% 이상 감소
- 재출동률 10% 이상 감소
- FTFR 8% 이상 상승
- MTTR 10% 이상 단축

## 10. 실행 원칙
- KPI 중심 운영
- 기능 추가는 근거 있을 때만
- 고객 피드백을 매주 제품에 반영
