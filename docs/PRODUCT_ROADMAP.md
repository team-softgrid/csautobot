# 전기차 충전소 AI 점검관리 서비스 로드맵

## 제품 비전

`csautobot`을 기반으로, 전기차 충전소 사후관리 업무를 지원하는
"데이터 기반 AI 점검/AS 코파일럿"으로 확장합니다.

- 기존 서비스: 설비 등록 + 점검표 입력 + 일지 전산화
- 목표 서비스: 장애 원인/조치/부품 추천 + 점검일지 AI 보조 + 운영 인사이트 자동화

## 단계별 로드맵

## Phase 1 (0~3개월): PoC 안정화

- 데이터 파이프라인 표준화: `ingest.py`, `build_index.py` 운영 절차 확정
- 하이브리드 검색 품질 개선: 장애코드/동의어 정규화 룰 고도화
- 현장용 답변 포맷 고정: 증상 요약, 원인, 점검 순서, 부품, 근거, 신뢰도
- KPI 측정 시작:
  - Top-1/Top-3 유사 사례 적중률
  - 1차 응대 시간 단축률
  - 엔지니어 수동 수정 비율

## Phase 2 (3~6개월): MVP 서비스

- 점검일지 AI 어시스턴트:
  - 텍스트/체크리스트/사진 업로드 기반 자동 초안 생성
  - 점검 결과 표준 코드화(원인 코드, 조치 코드, 부품 코드)
- 작업 워크플로우:
  - 점검/AS 요청 → AI 제안 → 엔지니어 확정 → 일지 저장
- 운영 대시보드:
  - 설비별 재발 장애 현황
  - 충전소별 위험도/우선 점검 대상

## Phase 3 (6~12개월): 상용화

- 멀티 테넌트 SaaS(운영사/협력사 분리)
- 권한/감사로그/데이터 보존정책 도입
- SLA 모니터링과 알림 자동화
- 월 과금 체계 적용(Free + Pro + Enterprise)

## Phase 4 (12개월+): 고도화

- 예지정비(재발 가능성 예측)
- 부품 수요 예측 및 재고 최적화
- 모델 라우팅(무료/유료 LLM 혼합)으로 원가 최적화
- 파트너 API(관제/ERP/그룹웨어 연동) 제공

## 핵심 성공지표

- MTTR(Mean Time To Repair) 감소율
- 재방문(동일 장애 재발) 감소율
- 점검일지 작성 시간 감소율
- AI 제안 채택률(엔지니어 승인 비율)
