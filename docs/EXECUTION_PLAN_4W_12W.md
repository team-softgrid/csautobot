# 전기차 충전소 AI 점검관리 서비스 실행계획

> 목적: 현재 `csautobot` 검색형 PoC를 4주 내 MVP 기반으로 정리하고, 12주 내 상용 전환 준비가 가능한 구조로 확장한다.

## 현재 기준선

- 현재 동작 범위:
  - `csData` 기반 AS 사례 인제스트
  - Chroma + BM25 하이브리드 검색
  - Streamlit 기반 유사 사례 질의응답
- 현재 부족한 범위:
  - 구조화 데이터 저장
  - 설비/충전소/점검일지 도메인 모델
  - 점검일지 AI 어시스턴트
  - 사용자/조직/권한/과금/감사로그

## 4주 실행계획

## Week 1: 데이터 기준선 확정

- 목표:
  - 4년치 CS/점검 데이터 통합 기준 정의
  - 원본/정제/인덱스 산출물 경로 분리
- 작업:
  - `csData/` 파일 분류 규칙 수립
  - `incident`, `action`, `part_usage`, `inspection_log` 기준 컬럼 맵 정의
  - 개인정보 마스킹 규칙 정의
  - 누락/예외 시트 리포트 자동화
- 산출물:
  - 데이터 사전 v1
  - 컬럼 매핑표
  - 마스킹 정책
  - 인제스트 예외 리포트
- 완료 기준:
  - 샘플 10개 파일을 동일 스키마로 변환 가능

## Week 2: 구조화 인제스트 구현

- 목표:
  - 검색용 JSONL만 만들던 파이프라인을 구조화 레코드 생성 파이프라인으로 확장
- 작업:
  - `ingest.py`를 `raw -> normalized -> search_docs` 3단계로 분리
  - `site`, `charger`, `incident`, `action`, `part_usage` 레코드 생성
  - `charger_id`, `site_id`, `occurred_at`, `reported_at` 추론 규칙 추가
  - 결과물을 JSONL 외에 CSV 또는 SQLite로 저장
- 산출물:
  - 구조화 데이터셋 v1
  - 검색 인덱스용 문서셋 v2
  - 데이터 품질 리포트
- 완료 기준:
  - 동일 장애의 설비/사이트별 집계가 가능

## Week 3: 점검일지 MVP

- 목표:
  - 점검/AS 요청 -> AI 초안 -> 엔지니어 확정 -> 저장 흐름 구축
- 작업:
  - 점검일지 입력 폼 추가
  - 체크리스트 + 메모 + 사진 경로 입력 지원
  - AI 요약/조치 초안 생성
  - 저장된 점검일지 조회 화면 추가
- 산출물:
  - Streamlit MVP 화면 2종
  - `inspection_log` 저장 구조
  - AI 초안 프롬프트 v1
- 완료 기준:
  - 엔지니어가 한 건의 점검일지를 저장하고 다시 조회 가능

## Week 4: 운영지표와 MVP 정리

- 목표:
  - MVP를 데모 가능한 수준으로 묶고 측정 체계 시작
- 작업:
  - Top-1/Top-3 적중률 수동 평가셋 구성
  - 점검일지 작성 시간 측정 로깅
  - 재발 장애 현황, 설비별 이력 요약 화면 추가
  - README/운영문서/실행 절차 정리
- 산출물:
  - MVP 데모 시나리오
  - 평가 리포트 초안
  - 운영 가이드
- 완료 기준:
  - 내부 데모와 파일럿 설명이 가능한 상태

## 12주 실행계획

## 1단계: 1~4주

- 범위:
  - 데이터 구조화
  - 점검일지 MVP
  - Streamlit 데모 정리
- 게이트:
  - 데이터 적재율
  - 검색 적중률
  - 점검일지 저장 성공률

## 2단계: 5~8주

- 목표:
  - 제품을 Streamlit PoC에서 서비스형 구조로 이동
- 작업:
  - FastAPI 백엔드 초안
  - SQLite/Postgres 전환
  - 인증 초안(운영사/협력사/관리자)
  - 테넌트 분리 키 추가
  - 사용량 로깅 추가
- 산출물:
  - API 명세 v1
  - DB 마이그레이션 초안
  - 서비스 아키텍처 문서
- 완료 기준:
  - 검색/점검일지 저장 기능이 API 기반으로도 동작

## 3단계: 9~12주

- 목표:
  - 상용 전환 준비
- 작업:
  - Free/Pro/Enterprise 권한 차등
  - 감사로그/활동로그
  - 월 사용량 집계
  - SLA 알림 초안
  - 월간 리포트 자동 생성
- 산출물:
  - 과금 지표 정의서
  - 운영자 대시보드
  - 파일럿 제안용 기능 목록
- 완료 기준:
  - 무료 배포 + 유료 전환 실험이 가능한 구조

## 우선순위

1. 데이터 구조화
2. 점검일지 저장
3. API/DB 분리
4. 인증/테넌트
5. 과금/로그

## 리스크와 대응

- 데이터 식별자 부재:
  - `site_id`, `charger_id`는 초기엔 규칙 기반 생성 후 점진 정제
- 사진 업로드 품질 문제:
  - 초기엔 파일 경로/URL 저장부터 시작
- 비용 통제:
  - Free 플랜은 검색 위주, 긴 생성 기능 제한
- 문서/코드 불일치:
  - 릴리스마다 운영 문서 업데이트 체크리스트 적용

## 권장 의사결정

- 4주 내에는 Streamlit MVP를 유지한다.
- 12주 내에는 FastAPI + DB 분리를 완료한다.
- 과금은 인증/사용량 로깅이 들어간 뒤에만 활성화한다.
